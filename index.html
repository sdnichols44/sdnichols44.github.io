Motorsport Manager String Compiler

Welcome!

Welcome to my first ever project! I have been learning both front and back-end web development for the past 6 months, and have created this simple web app to test my skills and help myself learn through working alone and pushing the boundaries of my (limited) knowledge. Note that looks weren't the aim here, it was more functionality and usefulness to an end user, and a test of my JavaScript ability.

I therefore present to you the Motorsport Manager String Compiler, a simple web app that allows modders of the PC version of Motorsport Manager to speed up some aspects of their work while also allowing new modders to get into making their own additions to the game. Currently all additions to areas such as Part Components, Personality Traits and Dilemmas require a rather large time commitment due to the manual nature of making additions, and can have quite a large chance for errors when converting the text files to csv and back again, which some have fallen victim to in the past.

The purpose of this app is to negate as many of these issues as possible by providing the means for a modder to define what they wish to add via inputs, which will then be arranged for them in the correct format to then be simply pasted into the text files, removing any need to convert files or manually type things out.

Personality Traits

Personality Traits are the way drivers take on buffs or de-buffs and add variety in the game. This is an area I've done a lot of work on in the past when adding new traits for Rebirth, and a large source of errors where even one thing out of place can cause crashes when trying to start a new game.

Here you'll be able to define all aspects of your trait using the inputs, and the result will be generated for you at the bottom of the page with the correct amount of commas to save you the hassle of counting them! The compiler will also allow you to activate fields for inputs that require you to make Frontend edits if you wish to add localisation to your traits.

Part Components

Part Components are the things the player uses to build their new parts during the season. While not as error prone as other areas, it is a rather large time sink when adding large quantities of new parts, with a lot of that time being taken up with writing up the flavour text that the player will see in game.

The compiler will automatically build the flavour text for you based on the effects of your component, and any bonuses it may carry, leaving you to just defining the rest. As with Personality Traits, you'll also have the option to activate fields that require Frontend edits if you wish to add localisation to your new components.

Make A Component!

Dilemmas

Dilemmas are the way the game tries to add some variability and action to the period between races and the off-season. We have been wanting to add some new Dilemmas to Rebirth for some time now, but the amount of time and work it takes to do so is quite prohibitive.

The compiler here is a little more substantial here as it requires the user to fill in multiple sections and link them appropriately. Dilemmas commonly come in four parts - header, body, button 1, and button 2. There is also sometimes a third button included at the author's discretion. This is also one of the ares of the game which does not require you to add anything to Frontend as all of the localisation fields are included in the file itself.

Make A Dilemma!

The above are just a few of the selection of string compilers I have created, you can find the rest in the dropdown menu on the navbar. More compilers are planned as I am able to get to them. Some of the more complicated areas may take longer, such as Interviews and Frontend, as they either have multiple sections, or are complicated for the user (or me!) to create. However, those that are available are fully functional and should help you as the modder to more efficiently and effectively add your content to the game.

I really hope you find this tool useful in your creation of additional content for Motorsport Manager, and please do let me know if you find any issues and have any problems using this tool.
